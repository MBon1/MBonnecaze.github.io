Independent Study Project (Pokémon Graph Application)

Engine: Unity

Languages: C#, SQL

Github Repository (Code Snippets)

The Pokémon Graph Application is a data visualization tool developed using the Unity 2D physics engine; programmed in C#. SQLiter was used to query Pokémon databases using SQL.

Nodes representing Pokémon types are spawned along the perimeter of the graph with Pokémon Nodes spawned between Type Nodes. A Pokémon Node has edges connecting them to these Type Nodes presenting the types of that Pokémon.

Feature: Spawn Types

To spawn Type Nodes along the perimeter of the graph, select the “Spawn All Types” button. Type Nodes are kinematic objects to keep them in place.
When producing a graph, this should be the first action done. Selecting the “Spawn All Types” button will clear all Pokémon Nodes if any have been spawned.

Function to Spawn Types Around Graph Perimeter: QuerySearchButtonHandler.cs/SpawnAllTypeNodes()

Feature: Set Pokédex

The Pokédex being used can be changed, which can allow us to limit our search to Pokémon available in a specific game. To change a Pokédex, select the dropdown underneath “POKÉDEX” and select the Pokédex you want to use. By default, the national Pokédex, which contains all Pokémon, is set. At the moment, we can select from the Pokémon Diamond and Pearl (DP), Pokémon Platinum (Pt), Pokémon Black and White (BW), and Pokémon Black 2 and White 2 (B2W2) Pokédexes.
Demo Timestamp

Pokémon Nodes

Pokémon Nodes are spawned at the midpoint between two Type Nodes. Spring joints are used to connect the Pokémon Nodes to their corresponding Type Nodes. Pokémon of the same type combination have collision enabled between them while Pokémon Nodes that do not have the same type combination will not. A slight attraction is placed on the Pokémon Nodes pulling them slightly towards the center of the graph through a point effector around the central point of the graph while an edge collider prevents the nodes from moving outside of the perimeter. When all Pokémon are spawned in, the Type Nodes resize relative to the number of Pokémon that have that type. Pokémon Nodes of the same type will group together in the same area of the graph.

There are various ways to query Pokémon and add them to the graph. You can select from Pokémon with a specific type combination, randomly select a given number of Pokémon, or select from all Pokémon in the current Pokédex. You can further modify the search parameters with the toggles and dropdowns to the right of the search buttons. These parameters allow you to, for example, search for Pokémon who do not evolve or exclude Legendaries and Mythical Pokémon.

Finding All Pokémon

Selecting the “Find All Pokémon” button spawns in all Pokémon from the selected Pokédex.

Find Random Pokémon

By selecting the “Spawn Random Pokémon” button, random Pokémon Nodes will be spawned into the graph. By default, 2 random Pokémon are spawned in. You can set how many Pokémon to be spawned by inputting a number into the textbox next to the button.
Demo Timestamp
Function to Perform Query: QuerySearchButtonHandler.cs/ExecuteRandomPokemonSearchQuery()

Find Pokémon by Type

Pokémon can be queried by type combination. You can set the desired type combination using the dropdowns beneath “TYPE 1” and “TYPE 2”. If either Type 1 or Type 2 are left blank, the application will search for Pokémon with only the one type inputted. The “...” option for Type 2 acts as a wild card. It will search for all Pokémon that have the selected type for Type 1 in its Type combination.

Feature: Query Parameters

Demo Timestamp

Search for Symmetric Typing

If “Search for Symmetric” is selected, the type search will look for Pokémon with that type combination and ignore the order of the types. By default, this option is selected.

Include Forms

When this toggle is selected, alternate forms are included when performing a query. This includes Mega Evolutions and Regional Variants.

Only Final Evolutions

When this toggle is selected, a condition is added to the query where only Pokémon who do not evolve will be spawned.

Not in Regional Dex

When this toggle is selected, a condition is added to the query where only Pokémon who are not a part of the selected Pokédex will be spawned.

Restricted Legendaries, Sub-Legendaries, and Mythicals

Certain Pokémon are categorized as Restricted Legendaries, Sub-Legendaries, or Mythicals. These Pokémon are typically powerful, late-game Pokémon that may be tied to an event. They tend not to be used by NPCs and are banned in some tournaments.
For each of these categories, you can select to “Include”, “Only” Include, or “Exclude” them from a query by selecting one of these options from their corresponding dropdown.

Clearing Nodes

Nodes can be cleared after a query. There are two parameters to select which nodes are deleted: “Clear Pokémon Nodes” and “Clear All Nodes”. Selecting “Clear Pokémon Nodes” will only remove the currently spawned Pokémon Nodes whereas selecting “Clear All Nodes” will also remove the spawned Type Nodes.

Working with the Resulting Graph

Resizing the Graph

The slider will change the radius of the graph, ranging between a radius of 14 and 140. You can see the size of the radius in the textbox next to the slider. When scaling the radius down, some Pokémon Nodes will remain on the outside of the graph. Selecting “Refresh Pokémon Pos.” button will take all Pokémon Nodes outside of the perimeter of the graph and move them back into the graph.

Zooming In and Out and Changing the Camera's Position

Using the scroll wheel of the mouse, you can zoom in and out of the graph. Pressing down the right mouse button and dragging it will move the camera's position in the direction of the mouse cursor. You can reset the camera by pressing the escape button.

Moving a Node

You can change the position of a node by left-clicking on it and dragging and dropping it to its new position.
Demo Timestamp

Fixing a Node's Position

Sometimes, nodes may overlap each other or a node may move a lot due to physics. Moving a node may not work as it will return to its previous position. To fix the position of a node, press the control key and left-click the node. This will freeze the X and Y positions of the node's rigid body.
To unfreeze a node, press the control key and left-click the node again.
Demo Timestamp
Code to Freeze Node's Position Locate in: NodeIcon.cs/BindNodeToConnecting()

Relative Positioning (Locking a Node's Position Relative to its Connecting Nodes)

You can force connecting nodes to stay a fixed distance from a “parent” node by pressing shift and left-clicking the parent node. Moving this parent node will also move its children nodes. You can lock both Pokémon Nodes to a Type Node or Type Nodes to a Pokémon Node.

To unlock the children nodes from their parent node, hold shift and left-clicking the parent node. You can manually invert this locked state of a node by holding down shift and right-clicking on a node. For example, if a child node is locked to a parent node, inverting its locked state will unlock the child node from its parent node.
